# Руководство по верификации водителей

## ✅ Что было добавлено

### 1. Функциональность верификации в диспетчерской

Теперь диспетчер может напрямую из интерфейса:
- ✅ **Верифицировать** водителей (одобрить)
- ❌ **Отклонить** водителей

### 2. Автоматическое обновление в реальном времени

- Данные водителей обновляются автоматически при изменении в Firebase
- Статус верификации отображается моментально
- Местоположение водителей на карте обновляется в реальном времени

---

## 🚀 Как использовать

### Шаг 1: Обновите правила безопасности Firestore

1. Откройте [Firebase Console](https://console.firebase.google.com/)
2. Выберите проект **TowTruckDriverAuth**
3. Перейдите в **Firestore Database** → **Rules**
4. **Скопируйте** содержимое файла `firestore.rules` из корня проекта
5. **Вставьте** в редактор правил
6. Нажмите **"Publish"**

**Важно**: Это обновит правила безопасности и разрешит диспетчеру обновлять статус водителей.

### Шаг 2: Перезапустите dev сервер

```bash
# Остановите текущий сервер (Ctrl+C)
npm run dev
```

### Шаг 3: Верифицируйте водителей

1. Откройте диспетчерскую в браузере
2. Перейдите на вкладку **"Водители"**
3. Найдите водителей со статусом **"Не проверен"**
4. Нажмите кнопку **"Верифицировать"** ✅
5. Статус водителя обновится на **"Проверен"** ✅

## 🗺️ Отображение водителей на карте

### Как это работает

1. **Мобильное приложение** отправляет геолокацию водителя в коллекцию `driverLocations`
2. **Диспетчерская** слушает изменения в реальном времени через Firebase `onSnapshot`
3. **Карта** автоматически обновляется при изменении местоположения

### Структура данных для отображения на карте

Для того, чтобы водитель отображался на карте, в Firestore должен быть документ:

**Коллекция**: `driverLocations`  
**ID документа**: `<UID водителя>` (тот же ID, что в коллекции `drivers`)

**Поля**:
```json
{
  "driverId": "H5aHIGs66fUZMF0prCEyQ5B2GVH3",
  "name": "Имя Фамилия",
  "lat": 55.7558,
  "lng": 37.6173,
  "status": "online",
  "updatedAt": <timestamp>
}
```

### Статусы водителей на карте

- 🟢 **online** - зеленый маркер (`/icons/marker-green.svg`)
- 🟠 **busy** - оранжевый маркер (`/icons/marker-orange.svg`)  
- ⚪ **offline** - серый маркер (`/icons/marker-gray.svg`)

## 🔧 Устранение проблем

### Водитель не отображается на карте

**Причина 1**: Нет документа в `driverLocations`

**Решение**:
1. Убедитесь, что мобильное приложение отправляет геолокацию
2. Проверьте в Firebase Console → Firestore Database → `driverLocations`
3. Если документа нет - мобильное приложение должно создать его при входе водителя

**Причина 2**: Неверные координаты

**Решение**:
1. Проверьте поля `lat` и `lng` в документе водителя
2. Координаты должны быть числами (не строками!)
3. Для Москвы: `lat: 55.7558, lng: 37.6173`

**Причина 3**: Устаревшие данные (старше 5 минут)

**Решение**:
- Диспетчерская считает данные устаревшими, если `updatedAt` старше 5 минут
- Водитель должен быть онлайн в мобильном приложении
- Мобильное приложение должно обновлять местоположение каждые 30 секунд

### Статус верификации не обновляется

**Причина**: Правила безопасности Firestore блокируют обновление

**Решение**:
1. Убедитесь, что вы обновили правила Firestore (см. Шаг 1)
2. Проверьте консоль браузера (F12) на наличие ошибок
3. Если видите ошибку "permission-denied" - правила не обновлены

### Кнопки верификации не появляются

**Причина**: Водитель уже верифицирован

**Решение**:
- Кнопки показываются только для неверифицированных водителей
- Проверьте поле `verified` или `isVerified` в Firebase для этого водителя

## 📊 Тестирование системы верификации

### Тест 1: Регистрация нового водителя

1. Зарегистрируйте нового водителя в мобильном приложении
2. Откройте диспетчерскую
3. ✅ Водитель должен появиться в списке с меткой **"Не проверен"**
4. ✅ Должны быть кнопки **"Верифицировать"** и **"Отклонить"**

### Тест 2: Верификация водителя

1. Нажмите кнопку **"Верифицировать"**
2. ✅ Появится уведомление "Водитель верифицирован"
3. ✅ Статус изменится на **"Проверен"** с зеленой галочкой ✅
4. ✅ Кнопки верификации исчезнут
5. ✅ Изменения видны в Firebase Console

### Тест 3: Отображение на карте

1. Откройте мобильное приложение на телефоне
2. Войдите как водитель
3. Включите геолокацию
4. Откройте диспетчерскую в браузере
5. Перейдите на вкладку **"Водители"**
6. ✅ На карте должен появиться маркер водителя
7. ✅ При движении водителя маркер обновляется в реальном времени

### Тест 4: Real-time обновления

1. Откройте диспетчерскую в двух вкладках браузера
2. В одной вкладке верифицируйте водителя
3. ✅ Во второй вкладке статус обновится автоматически (без перезагрузки)
4. ✅ Это работает благодаря Firebase `onSnapshot`

3. **Добавьте историю верификаций**
   - Кто и когда верифицировал водителя
   - Причина отклонения

4. **Уведомления для водителей**
   - Push-уведомление при одобрении
   - Push-уведомление при отклонении с причиной

## 📝 Примечания

- Все данные обновляются в реальном времени через Firebase Realtime
- Не нужно перезагружать страницу для обновления данных
- Геолокация обновляется автоматически мобильным приложением
- Диспетчерская подключается к Firebase через `onSnapshot` слушатели

**Поддержка**: Если возникли проблемы, проверьте:
1. Консоль браузера (F12) на наличие ошибок
2. Firebase Console → Firestore Database → данные
3. Firebase Console → Firestore Database → Rules
