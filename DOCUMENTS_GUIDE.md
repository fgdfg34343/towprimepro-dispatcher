# Руководство по просмотру документов водителей

## ✅ Что было добавлено

### Новая функциональность

1. **Кнопка "Посмотреть документы"** - добавлена для каждого водителя в списке
2. **Модальное окно с документами** - красивое отображение всех загруженных документов
3. **Просмотр в полном размере** - возможность увеличить любое изображение
4. **Скачивание документов** - кнопка скачивания для каждого документа
5. **Быстрая верификация** - можно верифицировать прямо из окна с документами

---

## 🎯 Как использовать

### Шаг 1: Откройте список водителей

1. Войдите в диспетчерскую
2. Перейдите на вкладку **"Водители"**
3. Найдите нужного водителя в списке

### Шаг 2: Просмотр документов

1. Нажмите кнопку **"📄 Посмотреть документы"** у нужного водителя
2. Откроется модальное окно со всеми загруженными документами
3. Каждый документ отображается с:
   - Типом документа (Водительское удостоверение, Паспорт и т.д.)
   - Статусом (На проверке / Одобрено / Отклонено)
   - Датой загрузки
   - Изображением документа

### Шаг 3: Детальный просмотр

1. **Увеличить изображение**: Нажмите на изображение документа
2. **Скачать документ**: Нажмите кнопку "📥 Скачать" под документом
3. **Закрыть окно**: Нажмите крестик или кликните вне окна

### Шаг 4: Верификация

Есть два способа верифицировать водителя:

**Способ 1**: Из окна документов
1. Просмотрите все документы
2. Нажмите кнопку **"✅ Верифицировать"** внизу окна
3. Документы закроются, водитель будет верифицирован

**Способ 2**: Из списка водителей
1. Закройте окно документов
2. Нажмите кнопку **"Верифицировать"** в карточке водителя

---

## 📋 Типы документов

Мобильное приложение водителя может загружать следующие типы документов:

1. **Водительское удостоверение** (`driverLicense`)
   - Фото или скан водительских прав
   - Обязательный документ

2. **Свидетельство о регистрации ТС** (`carRegistration`)
   - СТС автомобиля
   - Обязательный для водителей эвакуаторов

3. **Фото автомобиля** (`carPhoto`)
   - Фотография эвакуатора
   - Для проверки состояния транспорта

4. **Паспорт** (`passport`)
   - Основной документ удостоверяющий личность
   - Может требоваться для верификации

5. **Страховка** (`insurance`)
   - ОСАГО или КАСКО
   - Опционально

## 🖼️ Структура документов в Firebase

Документы хранятся в коллекции `drivers` в поле `documents`:

```javascript
{
  "documents": {
    "driverLicense": {
      "imageUrl": "https://firebasestorage.googleapis.com/...",
      "status": "pending",
      "uploadedAt": <timestamp>
    },
    "carRegistration": {
      "imageUrl": "https://firebasestorage.googleapis.com/...",
      "status": "approved",
      "uploadedAt": <timestamp>
    },
    "carPhoto": {
      "imageUrl": "https://firebasestorage.googleapis.com/...",
      "status": "pending",
      "uploadedAt": <timestamp>
    }
  }
}
```

### Статусы документов

- **`pending`** / "На проверке" - документ загружен, ожидает проверки
- **`approved`** / "Одобрено" - документ проверен и одобрен ✅
- **`rejected`** / "Отклонено" - документ отклонен ❌

## 🔧 Устранение проблем

### Кнопка "Посмотреть документы" не работает

**Причина**: Ошибка загрузки данных из Firebase

**Решение**:
1. Откройте консоль браузера (F12)
2. Проверьте наличие ошибок
3. Убедитесь, что правила Firestore разрешают чтение коллекции `drivers`

### Документы не отображаются

**Причина 1**: Водитель не загрузил документы

**Решение**:
- В окне документов будет написано "Документы не загружены"
- Водитель должен загрузить документы через мобильное приложение

**Причина 2**: Неверная структура данных в Firebase

**Решение**:
1. Откройте Firebase Console → Firestore Database
2. Найдите документ водителя
3. Проверьте наличие поля `documents`
4. Убедитесь, что структура соответствует примеру выше

### Изображения не загружаются

**Причина**: URL изображения недоступен или неверный

**Решение**:
1. Проверьте правила Firebase Storage
2. Убедитесь, что изображения загружены в Firebase Storage
3. Проверьте поле `imageUrl` в Firebase Firestore

## 📝 Примечания для разработчиков

### Расширение функциональности

Если нужно добавить новые типы документов:

1. Обновите массив `docTypes` в `DriverDocumentsDialog.tsx`:
```typescript
const docTypes = [
  { key: "driverLicense", name: "Водительское удостоверение" },
  { key: "newDocument", name: "Новый документ" }, // добавьте сюда
];
```

2. Мобильное приложение должно загружать документ с тем же ключом (`newDocument`)

### Изменение отображения

Компонент `DriverDocumentsDialog` полностью кастомизируемый:
- Можно изменить размеры изображений
- Добавить фильтры по статусу
- Добавить комментарии к документам
- Добавить историю изменений
