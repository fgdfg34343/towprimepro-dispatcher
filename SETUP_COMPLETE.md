## ⚠️ Что нужно сделать вам

### Шаг 1: Настроить Firebase конфигурацию

Откройте файл `.env.local` и замените placeholder значения на реальные данные из вашего Firebase проекта:

```env
# Эти значения нужно заменить:
VITE_FIREBASE_API_KEY=your-firebase-api-key-here           # ← ЗАМЕНИТЕ
VITE_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com      # ← ЗАМЕНИТЕ
VITE_FIREBASE_PROJECT_ID=your-project-id                    # ← ЗАМЕНИТЕ
VITE_FIREBASE_STORAGE_BUCKET=your-project.appspot.com       # ← ЗАМЕНИТЕ
VITE_FIREBASE_MESSAGING_SENDER_ID=your-sender-id            # ← ЗАМЕНИТЕ
VITE_FIREBASE_APP_ID=your-app-id                            # ← ЗАМЕНИТЕ
VITE_FIREBASE_MEASUREMENT_ID=G-XXXXXXXXXX                   # ← ЗАМЕНИТЕ (опционально)
```

**Где взять эти данные?**

1. Откройте [Firebase Console](https://console.firebase.google.com/)
2. Выберите ваш проект (или создайте новый)
3. Перейдите в **Project Settings** (иконка ⚙️)
4. Прокрутите вниз до раздела **Your apps**
5. Выберите веб-приложение или создайте новое
6. Скопируйте значения из `firebaseConfig`

### Шаг 2: Добавить ADMIN_UID (опционально, но рекомендуется)

Для ограничения доступа к диспетчерской только определенному пользователю:

1. В Firebase Console откройте **Authentication** → **Users**
2. Найдите пользователя диспетчера
3. Скопируйте его **UID**
4. Добавьте в `.env.local`:

```env
VITE_ADMIN_UID=скопированный-uid-пользователя
```

### Шаг 3: Создать тестовые данные водителей

Для того чтобы увидеть водителей на карте и в списке, нужно добавить данные в Firestore:

**Вариант A**: Следуйте инструкции в [FIREBASE_SETUP.md](./FIREBASE_SETUP.md#создание-тестовых-водителей)

**Вариант B**: Быстрый способ через Firebase Console:

1. Откройте Firestore Database в Firebase Console
2. Создайте коллекцию `drivers`
3. Добавьте документ:
```json
{
  "firstName": "Иван",
  "lastName": "Петров",
  "phoneNumber": "+79991234567",
  "vehicleType": "Эвакуатор",
  "isVerified": true,
  "createdAt": "2024-10-15T10:00:00Z",
  "updatedAt": "2024-10-15T10:00:00Z"
}
```
4. Скопируйте ID созданного документа
5. Создайте коллекцию `driverLocations`
6. Добавьте документ с **тем же ID**:
```json
{
  "driverId": "скопированный-id",
  "name": "Иван Петров",
  "lat": 55.7558,
  "lng": 37.6173,
  "status": "online",
  "updatedAt": "2024-10-15T10:00:00Z"
}
```

## 🔍 Устранение проблем

### Карта не загружается

**Откройте консоль браузера (F12)** и проверьте ошибки:

- Если видите ошибку о квоте API - проверьте лимиты в Google Cloud Console
- Если видите ошибку "InvalidKeyMapError" - проверьте правильность API ключа
- Убедитесь, что включен **Google Maps JavaScript API** в Google Cloud Console

### Водители не отображаются

**Откройте консоль браузера (F12)** и проверьте:

1. Есть ли ошибки Firebase? 
   - ❌ "Firebase: Error (auth/invalid-api-key)" → Проверьте VITE_FIREBASE_API_KEY
   - ❌ "Missing or insufficient permissions" → Настройте правила Firestore

2. Есть ли данные в Firestore?
   - Откройте Firebase Console → Firestore Database
   - Проверьте наличие документов в `drivers` и `driverLocations`

3. Правильно ли установлены переменные окружения?
   - Проверьте `.env.local`
   - Перезапустите dev сервер

### Не могу войти

1. Убедитесь, что пользователь создан в Firebase Authentication
2. Проверьте правильность email и пароля
3. Проверьте, что включен метод аутентификации Email/Password
