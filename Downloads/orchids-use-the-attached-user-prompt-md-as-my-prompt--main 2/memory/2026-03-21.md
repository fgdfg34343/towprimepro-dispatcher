## Dispatcher handoff + client notification

- Landing orders now persist a tracked `orderId` in `localStorage` and keep a live Firestore subscription to `orders/{orderId}`.
- Added a persistent site notification card for the latest order:
  - while dispatcher is searching for a driver
  - when a driver is assigned / accepts the job
  - with driver name, phone, truck type, and live status from the order document
- Booking modal success state now also upgrades in place when driver assignment data appears.
- Public Firestore access was narrowed to `get` on individual `towprime-landing` orders, while authenticated dispatcher UIs keep their normal access.
- Public order payload now includes compatibility fields (`clientName`, `clientPhone`, `pickupAddress`, `dropoffAddress`, `price`, `notes`) for dispatcher UIs.
